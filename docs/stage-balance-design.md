# Stage Balance Design

Machine Defense の新規ステージを作るときの設計図。
既存ステージは手動調整の蓄積があるため、当面はこの設計図で置き換えない。
新しく追加するステージ、または大きく作り直すステージだけ、この方針を参照する。

## 目的

- ステージごとの敵密度の穴を減らす。
- 難易度上昇を「敵数」「HP倍率」「敵種」「特殊イベント」の組み合わせで管理する。
- 手動調整の感覚を残しつつ、毎回ゼロからスポーン表を作らない。
- 後から見ても、なぜそのステージが難しいのか説明できる状態にする。

## 前提

現状のステージは参考にはなるが、完全な正解ではない。

- ボスは現状まだ弱く見える場面がある。
- 序盤ステージは、敵が出現地点で倒される時間がまだ発生しやすい。
- スポーン表は手動で作られているため、30秒単位で敵密度に穴が出ることがある。
- ブレード型のノックバックなど、特定キャラ構成で難易度が大きく下がる。
- ステージHP倍率は有効だが、HPだけでは緊張感を作れない。

そのため、この文書は「今のステージを模倣するため」ではなく、
今後のステージを安定して設計するための基準として使う。

## ステージ設計の基本単位

1ステージは、おおむね 5 分前後を基準にする。
ただし、序盤は体感テンポを優先し、実時間よりも「退屈な空白がないこと」を重視する。

ステージは 30 秒単位の区間に分けて見る。

- 0:00-0:30: 導入。敵は少なめでもよいが、すぐ交戦が始まる。
- 0:30-1:30: 最初の圧力。初期キャラだけでも処理できるが、放置で楽勝にはしない。
- 1:30-3:30: 主戦場。敵密度と敵種の組み合わせでステージの個性を出す。
- 3:30-4:30: 終盤圧力。押し返し・範囲攻撃・強化選択が効く時間帯。
- 4:30以降: 締め。ボス、精鋭、または大群でクリア直前の山を作る。

## 30秒ごとの確認項目

新規ステージでは、30秒ごとに以下を確認する。

- 敵が少なすぎる区間がないか。
- 敵が画面上端で即処理され続ける区間がないか。
- バリケード付近まで到達する敵が少しはいるか。
- 高速低HP、低速高HP、大群、特殊敵の役割が偏りすぎていないか。
- 経験値が極端に少なく、レベルアップが遅すぎないか。
- 逆に、レベルアップが連続しすぎて戦闘が止まりすぎないか。

目安として、プレイヤーが強化済みなら処理できるが、無強化・放置では押される状態を狙う。

## ステージ変数

新規ステージは、以下の変数を先に決めてからスポーン表を作る。

| 項目 | 役割 |
| --- | --- |
| `duration` | 最後の敵が出る時刻。基本は 270-330 秒。 |
| `enemyHpMult` | ステージ全体のHP倍率。難易度の土台。 |
| `maxCharacters` | 参戦可能人数。序盤の難易度調整に使う。 |
| `pressure` | 敵密度。敵数と出現間隔に反映する。 |
| `speedBias` | 高速敵をどれくらい混ぜるか。 |
| `tankBias` | 高HP・ノックバック耐性持ちをどれくらい混ぜるか。 |
| `swarmBias` | 大群系をどれくらい混ぜるか。 |
| `specialBias` | 回復、分裂、警告攻撃などの特殊敵をどれくらい混ぜるか。 |
| `boss` | ボスの有無、出現時刻、随伴敵。 |

## 敵の役割分類

敵IDは見た目ではなく、能力名として扱う。
後から外見をゾンビ、機械、異形などへ差し替えられるようにする。

| 役割 | 主な敵ID | 用途 |
| --- | --- | --- |
| 基本 | `basic`, `dense`, `light` | ステージの密度を作る。 |
| 高速低HP | `fast`, `runner`, `fragile`, `needle`, `scout` | リスキル防止、接近圧力。 |
| 大群 | `swarm`, `carrier` | 範囲攻撃の価値を出す。 |
| 高HP | `guard`, `brute`, `armored`, `crawler`, `core`, `mass` | 単体火力と押し返しの価値を出す。 |
| 破壊圧力 | `breaker`, `striker`, `charger` | バリケード前の緊張感を作る。 |
| 支援・特殊 | `medic`, `elite` | 後半や節目の変化。 |
| ボス | `boss_siege` | ステージの山場。随伴敵とセットで扱う。 |

## スポーンの作り方

スポーンは「常時圧力」と「山場イベント」に分ける。

### 常時圧力

敵がいない時間を作らないための薄いスポーン。

- 5-10秒ごとに少数の敵を出す。
- 序盤は `basic`, `light`, `fast` を中心にする。
- 後半は同じ役割でも `dense`, `guard`, `core` など高HP寄りに置き換える。
- 常時圧力だけで難しくしすぎない。山場イベントの下地として使う。

### 山場イベント

ステージごとの印象を作るスポーン。

- 大群をまとめて出す。
- 高HP敵を数体混ぜる。
- 高速敵を一時的に増やす。
- ボスに随伴敵を出す。

山場イベントの直後に完全な沈黙を作らない。
処理後も常時圧力が残っている状態にする。

## ボスステージ

ボスは単体HPだけで調整しない。
単体HPを上げすぎると、ただ硬い敵になりやすい。

ボスステージでは次をセットで設計する。

- ボス本体のHP倍率
- ボスのノックバック耐性
- ボスのスタン耐性
- ボス出現中の少量の随伴敵
- ボス前後の敵密度

現状の問題として、ボスが早く倒されすぎる場面がある。
今後のボスステージでは、ボスHPだけでなく、随伴敵の圧力も必ず見る。

目安:

- 序盤ボス: ボス1体 + 少量の基本敵
- 中盤ボス: ボス1体 + 高速敵または大群を少量
- 後半ボス: ボス1体 + 高HP敵または破壊圧力を少量

ボスステージは毎回入れない。
節目として、3、6、その後は5ステージ前後ごとに置く。

## 難易度上昇の優先順位

難易度を上げるときは、いきなり全項目を上げない。
以下の順番で調整する。

1. 敵がいない時間を減らす。
2. 出現地点で即処理される時間帯に高速低HPを混ぜる。
3. 30秒ごとの敵数を増やす。
4. HP倍率を上げる。
5. 高HP・耐性持ちを増やす。
6. 特殊敵を増やす。
7. ボスや大型敵の随伴敵を増やす。

HP倍率だけで難易度を上げると、テンポが悪くなる。
敵数だけで難易度を上げると、範囲攻撃が強くなりすぎる。
速度だけで難易度を上げると、防衛ラインの攻防が短くなりすぎる。

## 序盤ステージの方針

序盤は簡単でよいが、退屈であってはいけない。

- ステージ1-3は、ギリギリ感を作る。
- ただし負け続ける難易度にはしない。
- 初期キャラだけで処理できる敵と、少しだけバリケードに近づく敵を混ぜる。
- 参戦人数制限は有効。ステージ1は3体、ステージ2以降は4体、ステージ5以降は5体を基準にする。
- 初回レベルアップは早めに起こす。

序盤で避けること:

- 30秒以上、敵が出現地点で倒され続ける。
- 高HP敵だけでテンポを落とす。
- 高速敵だけで事故らせる。
- ボスを単体で置いて、ただ早く倒される。

## 中盤以降の方針

中盤以降は、プレイヤーがキャラ強化・装備・研究を使う前提にする。

- ステージ10以降は無強化クリアを想定しない。
- ステージ15以降は装備または研究がないと苦しい状態を作る。
- ステージ20以降はキャラレベル、装備、研究の複合成長を前提にする。
- ステージ25以降は編成の偏りが弱点として出るようにする。

ただし、特定キャラだけが正解になる状態は避ける。
ブレード型のノックバック、フィールド型の範囲持続、ビーム型の線攻撃などは、
敵側の耐性や敵種で価値が上下するようにする。

## 手動調整してよい例外

設計図は絶対ではない。
以下は手動調整してよい。

- ボス出現直後の随伴敵数
- 序盤30秒の体感テンポ
- 特定キャラが強すぎる、弱すぎる場合の敵種変更
- 背景や演出に合わせた敵の出現タイミング
- ステージ固有の見せ場

ただし、手動調整した場合も、理由をコメントまたは設計メモに残す。

## 将来の実装案

将来的には、完全な手書き `spawnTable` ではなく、次のような半自動方式にできる。

```ts
interface StageBlueprint {
  id: string
  duration: number
  enemyHpMult: number
  maxCharacters?: number
  pressure: number
  speedBias: number
  tankBias: number
  swarmBias: number
  specialBias: number
  boss?: {
    enemyId: string
    time: number
    addPressure: number
  }
  events: ManualSpawnEvent[]
}
```

この方式では、通常スポーンは設計図から生成し、ボスや見せ場だけ手動イベントとして追加する。
ただし、現時点ではまだ実装しない。
まずは新規ステージを作るときの設計メモとして運用する。

## 実装前チェック

新規ステージを実装する前に確認する。

- 30秒ごとの敵数に極端な穴がないか。
- ステージ最後の敵が遅すぎないか。
- ボスが単体で孤立していないか。
- 大群だけ、硬い敵だけ、高速敵だけになっていないか。
- 無強化放置で楽勝になっていないか。
- 強化済みプレイヤーでも、最低1回はバリケード付近の攻防が発生するか。
- クリア後に「次は強化しよう」と思える圧力があるか。
